# Nano-Bruce 项目使命

## Mission（使命）

Nano-Bruce 是一个**学习导向的极简 LLM Agent 框架**，致力于通过清晰、可运行的代码实现，帮助开发者深入理解 Agent harness 层的核心概念与实现方式。

我们相信：**理解比使用更重要**。与其提供一个功能完备但内部黑盒的商业产品，不如提供一条透明的学习路径，让每一位开发者都能"吃透" Agent 运行时的本质——对话循环、工具编排、上下文管理、流式事件处理。

## Target Audience（目标用户）

### 主要受众

1. **Agent 开发学习者**
   - 想要理解 LLM Agent 内部运作机制的开发者
   - 正在探索 Agent 架构设计模式的工程师
   - 需要可读、可调试代码作为学习材料的实践者

2. **独立开发者 / Solo Founder**
   - 需要快速搭建 Agent 能力的独立开发者
   - 希望在现有框架基础上进行定制化开发
   - 追求轻量级依赖、高可控性的技术选型者

3. **企业技术团队**
   - 评估不同 Agent 框架的技术选型者
   - 需要透明实现以便进行安全审计和合规检查
   - 希望在此基础上构建内部 Agent 平台

### 非目标用户

- 追求开箱即用、零配置体验的终端用户
- 需要企业级 SLA 保障的生产环境（当前阶段）

## Core Problems（要解决的核心问题）

### 1. 认知门槛问题

现有 Agent 框架（LangChain、AutoGPT 等）功能强大但抽象层级多，初学者难以理解底层机制。Nano-Bruce 通过**极简实现**降低认知负担。

### 2. 学习材料缺失问题

高质量、可运行的 Agent 实现参考稀缺。Nano-Bruce 提供**完整的、带注释的代码链路**，从消息处理到工具调用，每一步都清晰可见。

### 3. 框架锁定问题

商业框架往往绑定特定生态。Nano-Bruce 采用**Provider 抽象层设计**，支持 OpenAI、Moonshot、DeepSeek 等多模型，避免供应商锁定。

### 4. 定制化困难问题

大型框架难以魔改。Nano-Bruce 的**模块化设计**允许开发者只取所需组件，或替换任意层实现。

## Design Principles（设计原则）

1. **透明优先** —— 代码即文档，行为可追溯
2. **极简克制** —— 只做必要之事，避免过度抽象
3. **可组合性** —— 各模块独立可测试、可替换
4. **渐进式复杂度** —— 入门简单，深入有路径

---

**Why:** 明确项目定位，避免功能蔓延，确保开发决策有据可依
**How to apply:** 所有功能提案需对照使命声明，回答"这是否帮助目标用户更好地理解或使用 Agent？"
