# Design Docs Index

This folder stores architecture and design-level decisions.

## How to use

- Create one document per major design decision.
- Record context, options, decision, and consequences.
- Link related execution plans from `docs/exec-plans/`.

## Suggested template

1. Problem statement
2. Constraints
3. Options considered
4. Decision
5. Trade-offs
6. Verification approach
7. Rollback strategy
