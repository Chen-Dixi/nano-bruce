# Core Beliefs

## Repository as a recording system

- The repository is a memory system for agents and humans.
- Important decisions must be written, not implied in chat history.

## AGENTS as map, docs as source of truth

- `AGENTS.md` is concise navigation.
- Detailed knowledge belongs in structured `docs/`.

## Prefer explicit plans

- Non-trivial work should begin with an execution plan.
- Plans must include verification and risk notes.

## Reliability over speed

- Fast iteration is good, but silent failure is unacceptable.
- Agent work should be observable, reproducible, and debuggable.
